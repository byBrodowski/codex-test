Active Directory Federation Services (ADFS)

ADFS забезпечує автоматизовану ідентифікацію користувача та інтеграцію браузера із системою Windows для доменних облікових записів. Сервіс маркерів доступу відповідає за автентифікацію та зберігання даних користувача в зашифрованому вигляді.

Візуалізація процесу
Вбудована автентифікація
Підтримка Integrated Windows Authentication для безшовного входу без участі користувача.
Безпечні дані
Дані користувача зберігаються у зашифрованому вигляді та передаються для авторизації лише за потреби.
Сервер федерації
Генерує сертифікати автентифікації та перетворює заявки для ресурсних серверів.
Алгоритм авторизації
  1. Користувач звертається до веб-сервера, ADFS-агент перенаправляє на сервер федерації постачальника ресурсів.
  2. Відбувається виявлення домашньої області: користувач надає домен або email для визначення постачальника посвідчення.
  3. Сервер федерації ресурсів перенаправляє браузер на сервер федерації постачальника посвідчення.
  4. Користувач проходить автентифікацію з AD-обліковим записом.
  5. Після успішної перевірки створюється cookie-файл автентифікації та маркер безпеки ADFS.
  6. Маркер та cookie повертаються на сервер федерації ресурсів для трансформації заявок.
  7. Сервер федерації ресурсів формує новий маркер та передає його до ADFS-агента веб-застосунку.
  8. Веб-додаток інтерпретує заявки та надає користувачу веб-матеріали.
Домашній домен
Місце, де визначено та обслуговується посвідчення користувача (наприклад, fuib.com).
Результат
Безпечний та швидкий спосіб авторизуватися «в один клік». Всі операції відбуваються автоматично та без участі користувача.
